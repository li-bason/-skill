# 来源追踪规则

`source_manifest.json` 中每项拥有唯一 `source_id`。正式内容按
`source_id#页码/工作表/幻灯片/段落` 引用。

状态：

- `provided`：直接来自用户材料。
- `external`：来自联网补充，必须记录 URL、标题和访问日期。
- `inferred`：根据题型或原题推断，不是材料明示。
- `unverified`：暂时无法核验，交付时必须披露。

禁止：

- 未解析成功却声称“教材包含”。
- 为 PDF、图片或网页编造页码、标题或 URL。
- 把生成题标成原题。

材料冲突写入 `conflicts`，保留双方来源；默认按大纲 > 教师讲义 > 教材 > 用户笔记 >
外部资料排序，但不得悄悄删除冲突。

## 缺口回退链

当用户材料缺失、PDF 公式解析失败或现有内容不足以回答原子考点时：

1. 复核原文件的视觉页面、图片、表格或其他可用解析结果；
2. 仍有缺口时检索 Web，技术内容仅优先采用原始论文、标准、官方文档或权威教材页面；
3. 为网页建立独立 `source_id`，记录 `url`、`title`、`accessed_at`、`role: supporting`、
   `status: external` 和所补充的 `atomic_id`；
4. 正式资料中把外部补充与用户材料原文分栏或分段展示；
5. 不得以外部通用说法覆盖教师明确给出的考试口径。
