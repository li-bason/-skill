| 考点编号 | 考查要求 | 来源 |
| --- | --- | --- |
| `AP-P18-03` | 密度、r、minPts、核心点、直接可达、可达、异常点、优缺点 | `SRC-001 · Page 186, Page 189, Page 190, Page 191` |

#### 步骤与计算

1. 基本思想：以样本点在空间分布上的稠密程度为依据进行聚类，若区域中的样本密度大于某个阈值，则将相应的样本点划入与之相近的簇中；若密度小于一定程度则中断合并。
2. 关键参数：半径 r 和最小样本数 minPts。
3. 核心样本（Core Point）：以样本点 p 为圆心、r 为半径作圆，如果圆域内至少存在 minPts 个样本（包括 p 自身），则称 p 为核心样本，否则称为非核心样本。
4. 直接可达（Directly Reachable）：以核心样本 p 为圆心、r 为半径，圆域内的所有样本点对于 p 来说直接可达。直接可达是针对核心样本而言的。
5. 可达（Reachable）：如果存在样本点 p₁，p₂，…，pₙ，且 p=p₁、q=pₙ，pᵢ₊₁ 对于 pᵢ 来说直接可达，那么称 q 对于 p 来说可达；其中 p₁，…，pₙ₋₁ 均为核心样本。
6. 异常点（Outliers）：如果样本点 O 对于其他任何样本点来说均不直接可达，则称样本点 O 为异常点。
7. 优点：不需要输入类别数 K；可以对任意形状的稠密数据集进行聚类；可以在聚类的同时发现异常点；聚类结果稳定。

### AP-P18-04｜64. 层次聚类的流程、分类与应用

| 考点编号 | 考查要求 | 来源 |
| --- | --- | --- |
| `AP-P18-04` | 凝聚、分裂、树形结构、步骤 | `SRC-001 · Page 192, Page 193, Page 194, Page 195` |

#### 步骤与计算

1. 在数据挖掘和统计分析中，层次聚类分析（Hierarchical Cluster Analysis, HCA）又称系统聚类法，旨在得到样本簇结构的同时发现样本分布的层次结构。
2. 同时，层次聚类算法一般来说可以分为两种，一种是自下而上（Bottom-up）的凝聚层次聚类 / 聚集系统法（Agglomerative Clustering，常见）和自上而下的分裂层次聚类 / 分割系统法（Divisive Clustering，少用）。
3. 层次聚类法试图在不同层次上对样本集进行划分，进而达到形成树形的聚类结构。
4. 自下而上（Bottom-up）的凝聚层次聚类算法在聚类伊始会将数据中的每个样本点均看作是一个独立的簇结构，然后迭代将当前状态下最相似的两个簇进行合并，直到最后只剩下一个簇时聚类结束。
5. 对于自上而下的分裂层次聚类算法来说则恰恰相反，分裂层次聚类算法在聚类伊始将所有的样本点都看成是一个簇，然后迭代将当前状态下最大的簇划分为两部分，直到最后将整个数据集划分成n个簇结束。
6. 凝聚层次聚类具体步骤如下： （1） 输入样本集合、对聚类簇函数做出规定，给出聚类的簇数。
