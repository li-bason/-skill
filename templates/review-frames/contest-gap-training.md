# 竞赛补漏骨架 v1

## 提纲.md

```markdown
## <TOPIC-ID> 专题
### 已考点
#### <POINT-ID>｜<知识点>
> **考查要求：** <识记/推理/计算/应用>

##### <规则/方法/步骤/易错点/结论>
- ...（来源、置信度）
### 遗漏点
#### <GAP-ID>｜<遗漏点>
> **考查要求：** <识记/推理/计算/应用>

##### <定义/规则/解法/易错点/结论>
- ...（推断依据、置信度）
```

## 思维导图.md

使用 Mermaid `mindmap` 展示专题、已考点和遗漏点。

## 新题答案.md

每个三级及以下遗漏点标题下至少 3 题；每题使用 `question_id`，答案使用相同 `answer_id`。
题目与答案分别使用 `#### <ID>｜题目`、`#### <ID>｜答案`，不得用循环数字编号。
若包含选择题，题干及 A/B/C/D 之间保留空行，保证 Markdown 预览分段。
每个已考点和遗漏点均使用“考查要求＋分块内容”，不得只保留名称、来源或置信度。
并列规则和易错点一项一行；解法、证明和计算过程一步一行。
