# 期末复习流程

## 输入 GATE

先将用户在对话中直接给出的实质材料（尤其是课程名、大纲、题型和分值）原样登记到
`sources/`，再确认科目、教材/讲义、考试大纲、题型和考试日期。先建立课程与范围核对表：用户
明确给出的课程名/大纲为主来源；对每份材料核对标题、章节和课程归属，标为 primary、
supporting 或 excluded。不同课程材料不得合并成同一考试范围。每次只问一个问题。若没有正式
大纲，允许用户确认“按教材目录作为临时范围”；若题型缺失，必须确认题型后再出题。

## 分类

- `liberal-arts`：框架、概念、论述与材料题。
- `science`：定义、定理、推导与计算。
- `engineering`：知识点、方法、流程与应用。
- `language`：词汇语法、阅读、翻译和写作。

运行：

```bash
python scripts/task_manager.py init-final "<科目>" --category <分类>
python scripts/import_sources.py "<任务目录>" <材料...>
```

## 正式产物

文科、理科、工科：

- `atomic_points.md`：将大纲拆成可独立考查的最细节点；每个节点列出来源区间、考查动作和至少 3 题训练要求。
- `知识点.md`：章节 → 原子考点；每点固定使用“编号标题、考点信息表、内容区”。概念型使用
  `要点 | 内容` 表格；流程和计算型使用有序步骤与公式块。内容
  只摘录能直接回答该点的完整来源句，禁止复制整页或混入无关背景。
- `思维导图.md`：使用 Mermaid mindmap；内容必须与知识点树一致。
- `题目.md`：作为训练题库，按原子考点分组，每点至少 3 题，题目使用稳定 `question_id` 和 `atomic_id`。
- `答案.md`：与 `题目.md` 的 ID 和顺序完全一致。

必须先完成“全大纲覆盖审计”：大纲每个编号、每个并列短语、来源中“几个概念”实际展开出的
每个概念都映射到原子点。题库总题数下限为 `原子点总数 × 3`，而不是“大纲编号数 × 3”。
若来源在一个页面连续介绍多个方法、算法、任务或指标，必须按名称分成不同原子点，分别摘录；
不得为了减少标题数量而合并成一段。
同一原子点中的不同语义项也要分段展示。例如“目标”和“分类区别”分别使用独立引用段，
中间保留空行；清理 `Ø`、`•` 等幻灯片项目符号。
概念型最终显示为 `要点 | 内容` 表格，每个语义项一行；要点名称及关键术语加粗。
算法流程、实现步骤和计算过程不得使用表格，改用纵向有序列表，公式单独成块。
发现公式、上下标或希腊字母抽取异常时，渲染对应 PDF 页面核对视觉原文，再使用
Unicode 下标、Markdown/LaTeX 公式或清晰文字重新录入；不得保留问号占位和零宽字符。

语言类：

- `复习总结.md`：词汇语法 → 阅读 → 翻译 → 写作。
- `思维导图.md`：板块与策略总览。

## 题目策略

- 有官方题型分布时，将其作为题型比例与覆盖参考；它不减少每原子点至少 3 题的要求，也不新增未要求的产物。
- 无官方蓝图时，默认难度比例：基础 40%、中等 40%、综合 20%，按整套题近似满足。
- 优先使用用户提供的原题；不足部分生成补充题并标记 `generated`。
- 答案包含考点、思路、关键步骤和答案，不扩写成厚教辅。
- 外部补充只能用于明显缺口，必须标记来源 URL 和访问日期。
- 对材料缺页、公式无法辨认、概念说明不足等明显缺口，必须在视觉复核失败后检索权威 Web
  来源；技术内容优先原始论文和官方文档。外部内容单列为 `external`，不能冒充讲义原文。

## 完成条件

合同、来源 manifest、已登记的用户文本来源、课程与范围核对表、`atomic_points.md`、覆盖矩阵、思维导图和分类对应产物均存在；
每个原子点有能够直接回答该点的完整来源句和至少 3 道训练题；训练题/答案稳定 ID 完全同步；若有官方蓝图，训练题库的题型覆盖与其一致；校验器返回 0 error。
