## 快速体验

StudyForge 会根据用户的真实目标进入课程学习、考试复习、竞赛补缺或语言积累路线。

### 场景一｜期末复习

> **用户：** 期末。
>
> **StudyForge：** 好的，请告诉我课程名称。考试范围和课程资料也可以直接一起发来。
>
> **用户：** 离散数学。这是老师发的复习 PDF，没有教材。
>
> **StudyForge：** 我会先从复习 PDF 建立考试范围和原子考点，再检查资料是否只列名称、是否缺少定义与公式。缺失内容超过阈值时使用模型知识补全并标记来源状态，随后生成知识点、思维导图、训练题和答案。

这一场景会调用理科结构、数学符号修复、知识缺口补全和题答同步门禁。

### 场景二｜竞赛补缺

> **用户：** 帮我整理这几年的竞赛题，找出没有覆盖到的知识点。
>
> **StudyForge：** 我会先建立已考点和遗漏点清单，再围绕遗漏点生成补缺提纲、思维导图和新题答案；不同方法、指标和任务会拆成独立节点。

这一场景关注能力缺口，不会把作品提交型竞赛误套成刷题流程。

### 场景三｜语言积累

> **用户：** `account for` 是什么意思？
>
> **StudyForge：** 我会先检查个人词汇表，再给出当前语境下的英英释义、搭配、例句和近义词辨析，最后询问是否加入积累表。用户确认前不会修改文件。

语言路线与其他路线平级，是五类预设中的一个；它只维护词汇语法、翻译和英文句子三个积累表。

### 五类预设

| 分类 | 组织重点 | 典型产物 |
| --- | --- | --- |
| 文科 | 概念、背景、原因、内容、影响、论据、评价、结论 | 知识点、导图、论述训练、答案 |
| 理科 | 定义、定理、公式、推导、例题、结论 | 知识点、导图、计算与证明训练、答案 |
| 工科 | 概念、原理、参数、流程、伪代码、应用、注意事项 | 知识点、导图、计算与方案训练、答案 |
| 语言 | 词汇语法、翻译、英文写作句子积累 | 三个独立 Markdown 积累表 |
| 竞赛 | 已考点、遗漏点、能力缺口、补缺训练 | 提纲、导图、新题答案 |

### 四条应用路径

| 框架 | 解决什么问题 |
| --- | --- |
| 课程与认证学习 | 把期末、考研专业课、职业认证的范围、资料、知识点和训练题组织为完整闭环 |
| 教学与培训资产 | 把课程目标、讲义和能力要求转为可审计的知识结构与题库草案 |
| 语言三表积累 | 从单词查询、翻译和写作中持续沉淀个人语言资产 |
| 竞赛补缺训练 | 从历史题和能力要求中发现遗漏点并定向补练 |

不同路线共享来源追踪、增量更新、临时文件清理和质量门禁；内容结构与产物随任务改变。

## 核心创新

### 1. 目标先行，资料按需进入

考试大纲、课程目标、能力清单和教师要求拥有最高优先级。系统先把范围拆成能独立定义、比较、说明流程、证明或计算的原子知识单元，再去教材与课件中寻找内容。

### 2. 定向抽取，拒绝“整本资料搬家”

大型 PDF、PPTX 和教材默认只登记文件与元数据。建立考点映射后，`extract_relevant_pdf.py` 才会检索命中页和必要相邻页。

```text
考试范围
→ 原子考点
→ 关键词与同义词
→ 命中页面
→ 能直接回答考点的来源句
```

未命中页面不进入上下文，减少等待、无关文字和 Token 消耗。

### 3. 大纲拆到最细可考单位

“几个概念”“方法与步骤”“定义及计算过程”“优缺点”等复合表达会继续拆分。每个大纲编号及其并列子要求都必须映射到原子考点，覆盖矩阵负责反查遗漏。

### 4. 每个结果都有来源状态

| 状态 | 含义 |
| --- | --- |
| `provided` | 直接来自用户材料 |
| `external` | 来自登记完整的权威网页 |
| `inferred` | 模型补充，明确区别于教材原文 |
| `unverified` | 尚未可靠确认，交付时公开披露 |

来源页码、原子考点、知识点、题目和答案保存在内部证据链中；用户阅读的正文保持干净。

### 5. 同一骨架，适配不同学科

文科强调论据与评价，理科强调公式与推导，工科强调参数与流程。三类知识点都采用“考查要求＋分块内容”，目录、页眉、题型说明和同页其他考点不能进入正文。

所有并列要点逐项换行，证明、计算、论证和流程逐步展示。

### 6. 题目真正围绕当前考点

每道题绑定当前原子考点。选择题选项处于同一语义层级，干扰项来自真实易错点；其他章节标题和无关摘录不能随机充当选项。

同一考点的训练覆盖识记、理解和应用/计算，题目与答案的数量、ID 和顺序自动同步。

### 7. 资料不足也有清晰退路

系统先统计知识缺口。少量局部缺口可以检索少数权威来源；缺口超过阈值时停止逐站搜索，使用模型知识补充必要解释并标记为 `inferred`。

快速补全模式会向用户提示质量风险，教师大纲与用户资料仍保持最高考试口径。

### 8. 语言积累会识别、去重和联动

语言模块在写入前识别“已积累/未积累”。同一词条的新义项进入原分组，同一原句的新关键表达进入原翻译分组，重复写作句会被拦截。

内部 `language_index.json` 让一个词条能够反查相关翻译句和写作句；近义词辨析比较核心语义、语气强弱和当前语境可替换性。

<details>
<summary><strong>查看语言积累结构</strong></summary>

```markdown
## 词条：account for

| 词性 | 英文释义 | 搭配与用法 | 例句 | 同义词/近义词 |
| --- | --- | --- | --- | --- |
| phr v | to form a particular amount or part of something | account for + amount / proportion | Online sales account for 40% of the revenue. | constitute; make up |
| phr v | to explain the cause of something | account for + result / situation | Poor weather may account for the delay. | explain |
```

语言只使用纯 Markdown。词条、翻译原句和写作句分别作为自然去重键；没有实际关联作用的 ID 不会出现在表中。

</details>

### 9. 数学符号与乱码会阻止交付

Symbol、Wingdings、私用区字符、零宽字符、错位上下标和 `??` 公式片段会触发错误。系统要求回看原 PDF 视觉页面；仍无法确认的内容标记为 `unverified`。

### 10. 任务可以恢复，也可以继续更新

任务设计写入 `review_spec.md`，用户确认后锁定为 `review_lock.yaml`。来源、模板、输出列表、最低题量和进度均可恢复。

后续补充课件或修订范围时，只更新受影响部分，再重新执行覆盖和质量检查。

## 工作流

### 期末复习

```text
输入考试范围与课程资料
→ 核对课程归属
→ 拆分原子考点
→ 确认复习方案
→ 锁定任务合同
→ 按考点抽取资料
→ 生成知识与训练产物
→ 覆盖审计与视觉验收
→ Markdown / PNG / PDF 交付
```

### 竞赛补缺

```text
导入范围与历史题
→ 建立已考点
→ 识别遗漏点
→ 按缺口组织提纲
→ 生成补缺训练
→ 校验遗漏点覆盖
```

### 语言积累

```text
查询单词、翻译或写作句
→ 拆分词条与语义单位
→ 查询积累状态和跨表关联
→ 展示解释与辨析
→ 询问是否收录
→ 用户确认
→ 去重后增量写入
→ 重建内部索引
```

## 质量门禁

脚本通过不代表任务完成。正式交付还需要内容审查与视觉目检。

### 内容门禁

- 大纲编号和并列子要求是否全部覆盖；
- 知识点是否包含真实定义、公式、论据或步骤；
- 是否混入目录、页眉、题型说明和重复摘录；
- 题目是否只考当前原子点；
- 语言主项是否重复，写入是否获得用户确认。

### 结构门禁

- 每个知识点是否使用“考查要求＋分块内容”；
- 推导、论证、流程和计算是否逐步换行；
- 选择题选项是否逐项分段；
- 题目、答案和内部清单是否同步；
- 当前路线是否只生成合同声明的文件。

### 视觉门禁

Markdown 通过后仍需渲染 PDF 目检，检查公式、分页、表格、选项间距和思维导图。临时截图与 PDF 检查目录按需创建，验收后清理。

### 自动回归

| 测试 | 覆盖范围 |
| --- | --- |
| `test_golden_structure.py` | 黄金样例与核心模板结构 |
| `test_cross_route_regressions.py` | 五类路线的共享规则与隔离规则 |
| `test_language_accumulator.py` | 语言确认门、去重、增量更新与三表联动 |
| `gate.py` | 校验、回归、安全扫描和最终打包 |

## 参赛价值

本项目面向 [2026 中国高校计算机大赛—人工智能创意赛·鸿蒙高校创新赛](https://developer.huawei.com/consumer/cn/activity/incentive/C4)。它选择大学生熟悉且高频的学习场景，用可验证的 AI 工作流呈现真实落地价值。
