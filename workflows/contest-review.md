# 答题型竞赛复习流程

## 输入 GATE

确认比赛名称，并取得至少一套原题或模拟题。若有考纲，以考纲定边界、原题校验覆盖；
没有考纲时明确标记范围为“根据题集推断”。交论文或实操作品型比赛不得进入本流程。

运行：

```bash
python scripts/task_manager.py init-contest "<比赛名称>"
python scripts/import_sources.py "<任务目录>" <原题或考纲...>
```

## 正式产物

- `提纲.md`：按专题组织，分别列出已考点、遗漏点、来源和置信度。
- `思维导图.md`：专题 → 已考点/遗漏点的 Mermaid mindmap。
- `新题答案.md`：按遗漏点分组，每点至少 3 题，题后紧跟考点、思路和答案。

每个专题、遗漏点和题目使用稳定 ID。原题不默认复制进正式产物，只用于分析范围、
风格和难度；用户要求题库整合时才纳入。

## 补漏策略

- 新题贴近原题题干长度、能力层级、计算量和答案形式。
- 默认难度比例为基础 40%、中等 40%、综合 20%。
- 无法由原题支持的“遗漏点”标记为 `inferred`，不得写成确定事实。
- 外部资料只用于核对公开考纲或明显缺口，记录 URL 与访问日期。
- 大量资料缺失时不逐站联网；使用模型知识提出候选遗漏点、规则和解题方法，统一标记
  `inferred` 并提示核对。原题只用于范围与风格，不得把题型目录、试卷说明或其他专题答案
  随机拼进新题选项。

## 完成条件

提纲包含已考点与遗漏点；每个遗漏点至少 3 题；稳定 ID 唯一；来源、置信度和题目难度
可统计；校验器返回 0 error。
